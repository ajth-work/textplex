Implementation plan

1. Lock the pack format

Keep every language pack in a canonical lexicon.sqlite3 or lexicon.csv layout under resources/lexicon/<language>.

2. Build the Korean starter pack

Seed Korean first so the importer, lookup path, and preview surfaces have one non-Chinese reference implementation.

3. Repeat the same shape for the next languages

Use the same ingest and lookup rules for Japanese, Russian, Hebrew, and Arabic instead of creating one-off pipelines.

4. Anchor each pack to a learning benchmark

Track against TOPIK, JLPT, TORFL, and domestic literacy goals so vocabulary lists map to real learner expectations.

5. Keep the tracker visible in the app

Publish the status in the preview UI so the roadmap stays visible while the databases are still being assembled.

Current focus

Korean is the first live target because it forces the importer to handle Hangul, a new source-pack path, and TOPIK-aligned vocabulary at the same time.

The next pass after Korean should reuse the same pack contract for Japanese, Russian, Hebrew, and Arabic instead of creating separate code paths.
